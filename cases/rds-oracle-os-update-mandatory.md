# RDS for Oracle の必須 OS アップデートに関する問い合わせ

## ■ お客様からの問い合わせ内容

AWSより受領した「Amazon RDS for Oracleにおける必須のオペレーティングシステム更新の適用に関するお知らせ」について以下質問あり。

- 「自動的に新しい OS バージョンにアップグレードされます」と記載されているが、DBは停止されるのか
- 「2026 年 5 月 15 日よりも前にオペレーティングシステムをアップグレードすることをお勧めします。」と記載されているが、その意図について
- マネージドサービスにおけるRDS for OracleのOSとは何を指しているか
- 推奨されている作業、実施しない場合の影響

---

# ■ 調査内容

### ① RDS for Oracle の OS アップデート内容の確認
- AWS から送付された「必須のオペレーティングシステム更新」に関する通知内容を確認。
- 公式ドキュメントにて、OS アップデートは **約 10 分程度の停止を伴う** ことを確認。

 公式ドキュメント引用（[1] より）
> RDS for Db2、RDS for MariaDB、RDS for MySQL、RDS for PostgreSQL、および RDS for Oracle DB インスタンスでは、オペレーティングシステムのアップデートが必要になることがあります。
> Amazon RDS は、データベースパフォーマンスと顧客の全体的なセキュリティ体制改善のために、OS を新しいバージョンにアップグレードします。
> 通常、アップデートには約 10 分かかります。
> オペレーティングシステムのアップデートでは、DB インスタンスの DB エンジンのバージョンまたは DB インスタンスクラスは変更されません。

---

### ② 必須アップデートの意味・影響の調査
- 本アップデートは **必須（強制）更新** であり、2026/5/15 以降は AWS によりメンテナンスウィンドウ中に自動適用されることを確認。
- 強制適用時には **DB 再起動＝サービス断** が発生するため、非計画の停止リスクがある。

---

### ③ RDS の OS とは何かの説明整理
- RDS はマネージドサービスのため OS にはログイン不可だが、内部的には Oracle DB を動かすための OS が存在。
- 今回のアップデートはその **内部 OS へのセキュリティパッチ／バージョン更新** であり、適用は再起動によって実施される。

---

### ④ 推奨対応と未実施時のリスク整理
- 推奨作業として、顧客側で計画停止を確保した上で **2026/5/15 以前に OS アップデートを実施** することが望ましい。
- 実施しない場合のリスクとして、  
  - セキュリティリスク増大  
  - AWS による強制アップデート  
  - 予期せぬタイミングでの DB 停止 → サービス影響  
  を確認し、顧客に説明。

---

# ■ 回答（AWS公式ドキュメントに基づく）

- **DB 停止の有無**  
  RDS for Oracle の必須 OS アップデートに伴い、DB は再起動されるため停止します。アップデート自体は約 10 分程度です。

- **推奨実施時期**  
  2026 年 5 月 15 日より前に任意のタイミングで計画的にアップデートを実施することを推奨。

- **OS アップデートの内容**  
  RDS はマネージドサービスであるため OS に直接ログインは不可ですが、内部 OS 上で Oracle DB を稼働させています。今回のアップデートはその内部 OS に対するセキュリティパッチ／バージョン更新であり、ユーザー操作としては RDS インスタンスの再起動により適用されます。

- **アップデート未実施の影響**  
  - 古い OS に起因するセキュリティリスク増大  
  - 指定日以降、AWS による強制アップデートが実行される  
  - 想定外のタイミングで RDS 停止 → サービス影響の可能性
---

## ■ 最終結論（お客様側）

お客様側にて仕様を理解いただき、デフォルトイベントバス → カスタムイベントバスの転送設定を検討する との返信あり。
問い合わせクローズ。
